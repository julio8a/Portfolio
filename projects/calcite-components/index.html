Calcite Components

Calcite Components is a web component library built on Stencil JS and a foundational piece to the Calcite Design System.

Calcite Components GitHub repository

The problem

  1. The old framework that didn't support single-page applications, Calcite-Web (now deprecated).
  2. Calcite-Web was mainly used as a CSS framework. Therefore, for complex components such as sliders, date-picker, color-picker, etc. There was no unified solution or component.
  3. There was no JS framework-agnostic solution. Teams were creating their own React, Ember, Vue, Angular, components and were not built to share.
  4. There was no centralized location for sharable, components.
  5. High development cost due to teams rebuilding components each time they are needed (siloed teams)
  6. Inconsistent family of products (UX/UI pattern inconsistency) since teams were always trying to reinvent the wheel.

Goals for Calcite Components

  1. Configurable and easy to components
  2. A consistent system of components
  3. Open source building and collaboration
  4. Variance and flexibility within a component. Use props to change theme, color, size, spacing, etc.
  5. (Sketch, XD, Figma) Libraries for designers to quickly mock-up. 100% parity to the calcite-components repository

Component Life Cycle

We have an open and transparent process on GitHub.





There are two crucial pieces that made Calcite Components a great success from this life cycle:

  1. From the assignment, a design system team member always reviews (or creates) the requested component. We also make sure it stays within the current system. Consistent hover, active, focus, font size, font-weight, etc.
  2. The implementation can be done by the requestor (We have documentation on consistent API nomenclature and component structures so that new things are consistent regardless of who builds it. The core team can build it, or any other contributing member.

What comes out of the box with Calcite Components (code and design)?

  1. All components pass accessibility compliance
  2. Right to Left support (RTL, i.e. Arabic)
  3. Internationalization (i.e. date picker displays date order based on location, 25/12/2020)
  4. Light and Dark theme
  5. We leverage shadow dom, so anybody using our components won't have any style clashes
  6. Exposed CSS variables for theming
  7. Framework agnostic. Doesn't matter what JS Framework you're using, you can easily consume these components (ember, angular, react, etc)
  8. Flexible through props and components can be sized. i.e. size="s,m,l,etc"
