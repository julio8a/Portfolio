This is an executive dashboard I created for managers, executives and CTOs of Esri. It is the centalized location for all Programs I manage as the Design Program Manager. It holds key high level infomration to keep stakeholders informed and the status of all programs.

Need for this project

When I started my new role as Design Program Manager in 2018, there were not processes or tools in place for managing or directing programs. Moreover, no tools to keep management informed of the progress. The purpose of this dashboard is to keep management, as well as everyone at the company, informed about Calcite programs. The objective of Calcite is to create a centralized design system and language, which will enable standardization, consistency, and harmonization through the ArcGIS platform and all other Esri initiatives.

Requirments and features

  • Overview of resrouces allocated to each program and overall percentage allocation
  • Display number of outstanding, in progress, and completed tasks per individual program
  • Design and development phase of individual programs
  • Percentage completion of individual programs
  • List of internal and external stakholders for each program
  • List of designers and developers working on each program
  • iframe of prototypes, builds, designs or invision prototypes
  • Interactive so that you can jump between programs and quickly get the high level statistics

Tools used

There are quite a few of data visualization and dashboard solutions out there. Moreover, since the data I'm trying to display isn't geographic data, there are no restictions to using Esri software. However, in attempts to show colleagues and partners at the company that our software can be used with non-geographic datasets, I leveraged Operations Dashboard for ArcGIS for this project. Though there were some chanllenged is the way my dataset was structured, I was able to successfully build out my dashboard.

I also needed to integrate issues from Github projects. I was able to export a JSON file of all the github issues, dump it on a web server and use python to parse the issues and categorize them appropriately. From there, they got pushed to a hosted table on ArcGIS Online and I wa able to import that layer into my dashboard.
